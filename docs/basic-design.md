# 基本設計書: Raw Image Viewer

## 1. 目的

本書は、VS Code 拡張機能 Raw Image Viewer の基本設計を示す。
対象は、利用者向けの提供価値と主要な構成要素、および安定して維持すべき仕様の要点である。

実装詳細、メッセージの厳密仕様、エラー処理の詳細は [詳細設計](detailed-design.md) を参照する。
将来要件は [ロードマップ](roadmap.md) を参照する。

## 2. 対象範囲

現行実装で提供する主要機能は以下。

- raw 画像専用拡張子（`.raw`, `.gray`, `.yuv`）の自動オープン（`customEditors` priority: `default`）
- 汎用バイナリ拡張子（`.bin`, `.data`, `.img`）はビューア対象だが自動オープンしない（priority: `option`）。ファームウェアやディスクイメージなど raw 画像とは限らないファイルを誤って乗っ取らないための区別
- 任意ファイルを Open as Raw Image で手動オープン（`.bin` / `.data` / `.img` を開く標準的な方法。エディタの Open With... からの再オープンでも可）
- `.rawimagerc` / ファイル名推論 / ワークスペース設定によるレンダリング条件解決
- Webview + Canvas による画像表示
- PNG エクスポート
- ファイル更新時の再描画

## 3. アーキテクチャ概要

本拡張は 2 層構成で動作する。

| 層             | 役割                                               |
| -------------- | -------------------------------------------------- |
| Extension Host | ファイルアクセス、設定解決、Webview 連携、保存処理 |
| Webview        | 画像デコード、Canvas 描画、UI と操作処理           |

層間通信は postMessage ベースの双方向メッセージで行う。

## 4. 基本設計方針

### 4.1 設定解決の方針

レンダリングに必要なパラメータは次の優先順位で解決する。

1. `.rawimagerc`
2. ファイル名推論
3. ワークスペース設定

これにより、プロジェクト単位の明示設定を優先しつつ、設定不足時にも実用的な表示を維持する。

### 4.2 表示品質と操作性

- 複数ピクセルフォーマットを同一 UI で扱う
- 画面表示は Canvas を用いて効率的に描画する
- エクスポートは画面表示内容を基準に PNG 保存する

### 4.3 安全性と制約

- Webview では CSP を適用する
- 参照可能リソースは必要最小限のディレクトリに制限する
- Webview のレンダリングロジックは独立したコンパイル単位（TypeScript）として実装し、拡張本体と同じ型検査・Lint の対象とする。バンドル済みアセットとして nonce 付きスクリプトタグから読み込む
