# ロードマップ: Raw Image Viewer

## 1. 目的

本書は Raw Image Viewer の将来要件と拡張計画を管理する。
未実装機能の候補、期待される利用価値、段階的な実現方針を記載する。

現行実装の設計仕様は [基本設計](basic-design.md) と [詳細設計](detailed-design.md) を参照する。

## 2. 実装済み機能

以下は当初ロードマップに記載していたが、現在の実装に含まれている。

### 2.1 インタラクティブ閲覧（Zoom & Pan）

- Ctrl+スクロールによるズーム（1% 〜 3200%）
- ドラッグによるパン
- Fit ボタン（画面フィット）、1:1 ボタン
- ビューポートリサイズ時の自動フィット（ResizeObserver）

### 2.2 ピクセル情報インスペクション（Data Probe）

- マウスオーバー時のカーソル座標（x, y）表示
- フォーマット別の生ピクセル値表示
  - RGB/BGR: R・G・B チャンネル値
  - グレースケール・float32・depth16: Gray 値（float は小数点以下 4 桁）

### 2.3 ウィンドウ/レベルスライダー

- gray8 / gray16le / gray16be / depth16 / float32 向けの Min・Max スライダー
- 自動検出した値域を初期値に設定
- Reset ボタンで初期値に戻す

### 2.4 フォーマットオーバーライドドロップダウン

- Webview ヘッダーの `<select>` から `.rawimagerc` を変更せずに表示フォーマットを切り替え可能
- `supportedFormats`（`types.ts`）をドロップダウンの選択肢として利用
- 新しい `render` メッセージ受信時はオーバーライドをリセット

## 3. 将来要件

### 3.1 補間方法の切り替え

- Nearest Neighbor（ピクセル確認向け）
- Bilinear（視認性向け）
- 現状は `image-rendering: pixelated` 固定

### 3.2 リアルタイム解析

- ヒストグラム（輝度またはチャンネル別）
- 統計表示（最小値、最大値、平均値、中央値）

### 3.3 動的 UI コントロール

- ~~フォーマット切り替え~~（2.4 として実装済み）
- Webview 上からのライブ設定変更（幅、高さ、ヘッダーサイズ）
- マルチバイトフォーマットのエンディアン即時切替

### 3.4 Data Probe の拡張

- YUV フォーマット向けの生 YUV 値表示（現状は変換後の RGB 値のみ）

## 4. 段階的な実装案

1. 補間切替の追加
   - Nearest Neighbor / Bilinear の切り替え UI を実装し、表示品質の選択肢を増やす

2. 解析機能の追加
   - ヒストグラムと統計情報を導入し、解析ユースケースに対応

3. ライブ設定の統合
   - Webview 上でパラメータを直接編集できるようにし、設定の試行錯誤を高速化

## 5. スコープ外

本書は計画管理を目的とし、現行実装の正確な挙動定義は扱わない。
実装仕様は詳細設計で管理する。
